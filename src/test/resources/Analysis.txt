C помощью утилиты jstat установлено, что при добавлении, изменении,
получении заявок из базы данных память изменяется в области памяти EU.

При анализе работы приложения с помощью jconsole (анализ графиков) видно,
что при активном добавлении заявок в базу данных резко увеличивается
затрачиваемая память (Heap memory usage и CPU Usage).
При удалении заявок память также затрачивается (Heap Memory Usage)
пропорционально добавлению заявок в БД. CPU Usage также
локально расходует память, после чего приходит в исходное состояние.
При добавлении 100 заявок в базу данных произошла очистка мусора.
Уменьшилось количество объектов и количество занимаемой памяти (jmap).
После сборки мусора (или в процессе добавления новых заявок ва БД) стали
использоватся области памяти которые раньше не задействовались (jstat).
После очистки мусора Heap Memory Usage резко падает до первоначальной позиции (jconsole).
